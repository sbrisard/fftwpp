namespace fftw¶

C++ bindings to the FFTW library.

fftwpp strives to remain as close as possible to the orginal C library, while being more idiomatic. The user should refer to the documentations of the FFTW library for a more in-depth description.

class Plan¶
#include <fftwpp.hpp>

A wrapper class around fftw_plan.

The user is referred to section 4.2, Using Plans, in the FFTW documentation (http://fftw.org/fftw3_doc/Using-Plans.html#Using-Plans).

This class implements the RAII paradigm: the underlying fftw_plan is released when the wrapper object falls out of scope.

The constructor of this class should in general not be called directly. Rather, use a PlanFactory.

class PlanFactory¶
#include <fftwpp.hpp>

This factory class is used to create new instances of Plan.

Planner flags (see section 4.3.2, Planner Flags, in the FFTW documentation, http://fftw.org/fftw3_doc/Planner-Flags.html#Planner-Flags) are set through set_XXX()/unset_XXX() methods.

This class exposes a fluent interface, which allows to chain set_XXX()/unset_XXX() methods like so

auto plan = factory.set_estimate().set_preserve_input().create_plan();
